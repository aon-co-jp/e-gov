# e-gov.info (العربية)

مشروع يوحّد بين (1) الحكومة الرقمية (إجراءات إدارية بلا أوراق) و(2) منصة
تجارة إلكترونية وعقارية للأفراد وصولاً إلى الشركات التجارية، ويمكن الوصول
إليها عبر مداخل متعددة: تطبيق LINE، وموقع إلكتروني، وأجهزة وسائط متعددة في
متاجر الراحة.

> ⚠️ **هذا لا يزال مجرد نموذج/عرض توضيحي فقط — وليس خدمة فعلية.**
> يظهر هذا التنبيه في كل مرة، سواء استخدمت الموقع الإلكتروني أو أضفت حساب
> LINE كصديق. التوثيق الإلكتروني والعقود الإلكترونية الملزمة قانونياً (بيع/
> إيجار العقارات) لم تُنفَّذ بعد، بانتظار الموافقة التنظيمية الرسمية.

## ركيزتان

### 1. الحكومة الرقمية

- **مداخل متعددة**: تطبيق LINE وموقع إلكتروني، بالإضافة إلى أجهزة وسائط
  متعددة في متاجر الراحة (Loppi، Fami Port، آلات النسخ متعددة الوظائف في
  7-Eleven) لمن ليسوا معتادين على المواقع الإلكترونية أو الهواتف الذكية.
- **التحقق المتدرج من الهوية**: تتصاعد قوة التحقق مع قيمة/أهمية المعاملة
  (تحديد هوية المتصل + معاودة الاتصال ← رمز مرور لمرة واحدة عبر البريد
  الإلكتروني ← مسح NFC لبطاقة My Number عبر الهاتف الذكي).
- يهدف إلى طلبات إلكترونية بلا أوراق وتجربة "نافذة واحدة" عبر عدة جهات
  حكومية/بلديات محلية.

### 2. منصة التجارة الإلكترونية (التجارة عبر الدردشة بالذكاء الاصطناعي)

- سوق عام، من المواد الغذائية إلى السيارات ومعدات الصوت، يمكن اكتشافه
  وطلبه عبر محادثة مع الذكاء الاصطناعي على LINE والويب معاً.
- نموذج سوق للأفراد وصولاً إلى الشركات التجارية، برسوم أقل بكثير من
  المنصات الكبرى الحالية.
- **تقييم ائتماني بالذكاء الاصطناعي، ائتمان تجاري، وضمان الذمم المدينة**:
  شراء بالدفع الآجل استناداً إلى درجة ائتمان يحسبها الذكاء الاصطناعي،
  وكشف الفواتير المكررة، وضمانات الذمم المدينة.
- **الاستثمار العقاري ومكتب البناء بالذكاء الاصطناعي**: يقترح الذكاء
  الاصطناعي مخططات أرضية بناءً على معلومات الأرض التي تم البحث عنها، مع
  مبدأ تصميمي يقضي بعدم تغذية تدفق رؤوس أموال مضاربة مفرطة إلى العقارات.
- تعمل المرحلة الأولية دون مخزون فعلي، وتموَّل عبر الإعلانات كعرض
  توضيحي لتجربة المستخدم في التجارة عبر الدردشة بالذكاء الاصطناعي.

## النماذج المرجعية المدروسة

- إستونيا e-Estonia / X-Road (بنية تحتية لامركزية لتبادل البيانات)
- أذربيجان ASAN xidmet (مراكز خدمة بنافذة واحدة، وحدات متنقلة)

## البحث والتسويق الآليان

يجري بحثاً دورياً حول مواضيع الحكومة الرقمية باللغتين اليابانية
والإنجليزية (استدعاءات فعلية لواجهة برمجة تطبيقات بحث GitHub، وروابط بحث
Google مُولَّدة)، ويُعِدّ دورياً مسودات تسويقية لكل ميزة (انظر صفحة
`/research`).

## التكامل مع LINE

بالإضافة إلى تصفح الموقع الإلكتروني، يمكنكم إضافة حساب LINE الرسمي
كصديق وطرح الأسئلة عبر الدردشة (رد تجارة دردشة بالذكاء الاصطناعي قائم
على القواعد، يُستدعى بكلمات مفتاحية مثل "طلب"، "شراء"، "ائتمان تجاري"،
"البحث عن أرض"). سيُنشر رمز QR لإضافة الصديق بمجرد إصدار المعرّف
الأساسي (Basic ID) لحساب LINE الرسمي.

## العرض التلقائي من GitHub

إلى عرض بأسلوب `.rs`).

## حزمة التقنيات

Rust + [Poem](https://github.com/poem-web/poem). لا يعتمد على قاعدة
بيانات، ملف تنفيذي واحد مكتفٍ ذاتياً. نفس حزمة التقنيات المستخدمة في
`aruaru-tokyo`/`karu.tokyo`.

راجع [CLAUDE.md](CLAUDE.md) للاطلاع على فلسفة التصميم الكاملة.
